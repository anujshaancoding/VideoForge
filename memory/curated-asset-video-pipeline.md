CEO direction (2026-06-28): for script→video, **source real assets instead of AI-generating** them — AI hallucinates named/branded subjects (a "Valorant Jett" comes out generic). Build a reusable **Asset Library → retrieval-first pipeline → AI-gen fallback → template-driven layouts per video type**. The retrieval seam already exists: [[script-studio-sketch-images]] (`apps/api/src/script/imageSearch.ts` findSceneImage → `imagegen.ts` fallback).

**Three niches, in priority order:**
1. **Valorant (LEAD — audience buildup, long-term).** Formats: comparison / choice-tier-list / review (new agent/map/bundle). Asset source = **valorant-api.com** (free, official Riot art + structured stats), NOT Google/Pinterest scraping. Legally cleanest via Riot Fan Content Policy.
2. **Power BI / data-viz (credibility + product funnel).** Assets = CEO's OWN screen recordings of the 03_ZENTRIX visuals (vizHUB / powerbi-visuals) → zero legal risk, highest quality. Needs a capture pipeline. Plus generic tips/news.
3. **Fashion (later).** Don't download yet. Sources when ready: Pexels/Unsplash (free-commercial), brand press kits/lookbooks, affiliate product feeds. AVOID Pinterest/Vogue (copyrighted).

**CEO decisions (2026-06-28):** Valorant first · **9:16 Shorts/Reels** (matches current pipeline) · **audience-first, monetize later** → avoid copyrighted scraping, stamp provenance/license on every asset now so sourcing is auditable before any monetization.

**Sourcing legality (the real gate):** Google Images / Pinterest / Reddit = copyrighted + TOS-violating → ❌. valorant-api (Riot fan policy) ✅, own screen recordings ✅, Unsplash/Pexels/Pixabay ✅.
